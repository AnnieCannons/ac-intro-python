What is Python?

  • High-level computer language
  • Compiler: a program which takes in a different high-level computer program as input and outputs the same program but written in machine language
  • Interpreter: a program which imitates a computer that understands a high-level language

Elementary Data Types: Strings

  • A list of characters
  • Denoted by single or double quotation enclosure
  • Examples:
    • "hello world!"
    • 'I am 20 years old.'

Elementary Data Types: Integers

  • A whole number (i.e. no decimal)
  • Examples:
    • -14
    • 0
    • 9
  • Note: there is no such thing as infinity in Python

Elementary Data Types: Floats

  • Floating point real values
  • Decimal numbers
  • Examples:
    • 4.5
    • -8.0
    • 0.0
  • Note: In JavaScript there is only one number data type. In Python there are TWO!

Elementary Data Types: Booleans

  • True or False
  • Must be capitalized
  • NOT enclosed in quotes --> different from strings!
  • Examples:
    • True
    • False

Tip: To check the type of any given piece of data use the type() functioin --> type(4) returns int, type("h") returns str, etc.

Elementary Operators: +

  • Addition
  • Applicable to strings, ints, floats
  • String addition --> concatenation
  • Examples:
    • “hello” + “world” --> “helloworld”
    • 1 + 2 --> 3
    • 8.0 + 2.0 --> 10.0

Elementary Operators: -

  • Subtraction
  • Applicable to ints, floats
  • Examples:
    • 4 - 3 --> 1
    • 5.25 - 4 --> 1.25

Elementary Operators: *

  • Multiplication
  • Applicable to strings, ints, floats
  • String multiplication is called repeated concatenation
  • Examples:
    • “hello” * 3 --> “hellohellohello”
    • 3 * 1.5 --> 4.5

Elementary Operators: /

  • Division
  • Applicable to ints, floats
  • Examples:
    • 4 / 2 --> 2
    • 8 / 3 --> 2.666

Elementary Operators: //

  • Floor division
  • Returns the rounded down result when two numbers are divided
  • Applicable to ints, floats
  • Examples:
    • 3 // 2 --> 1
    • 10 // 6 --> 1

Elementary Operators: %

  • Modulo
  • Returns the remainder when two numbers are divided
  • Applicable to ints, floats
  • Examples:
    • 3 % 2 --> 1
    • 10 % 6 --> 4

Elementary Operators: **

  • Exponent
  • Applicable to ints, floats
  • Examples:
    • 2 ** 3 --> 8
    • 5 ** 0 --> 1

Note: PEMDAS still applies to all expressions.

Variables

  • An identifier that stores a value
  • Can represent any data type
  • Assign variable names to values by using =
    • = does not mean "equal" --> means "set to"
  • Note: all variables must be assigned values when they are declared
    • 					
					myNum = 10
					myDog = "Coach"
					myBool = True

Variables: Naming Rules

  • Must start with a letter or underscore
  • Can include numbers, letters, underscores
  • Case sensitive (i.e. myNum does not equal mynum

Variables: General Rules

  • Variables only remember the most recent value assignment 
    • 						
						myName = "Annie"
						myName = "Cannons"
  • myName now holds the value "Cannons"

Variables: General Rules

  • Variables can be set to themselves
    • 						
						totalAmount = 30.25
						totalAmount = totalAmount + 2
  • totalAmount now holds the value 32.25
  • Works because the computer first processes the right hand side of the equal sign and then the left
  • The following two lines have the exact same meaning:
    • 						
						totalAmount = totalAmount + 2
						totalAmount += 2

Variables: General Rules

  • Simultaneous assignment --> assign multiple variables at once
    • 						
						x, y = 30, 20
  • x holds the value 30
  • y holds the value 20

Converting Data Types

  • Let myAge = "46"
  • myAge is a string
  • What if you want it to be an integer? A float?
  • Use built-in functions to change the type of a variable!

Converting Data Types

    						
						myAge = "46"
						myAge = int(myAge)
						myAge = float(myAge)
						myAge = str(myAge)

    In this example, the variable myAge starts off as a string, then is changed to an integer data type, then to a float data type, and finally back to a string data type.

Converting Data Types: Rules

    						
						int("hello")
  • This will result in an error because the characters are not numbers
  • General rules for converting:
    • An integer can always become a float or a string
    • A float can always become an integer or a string
    • A string can sometimes become an integer or a string depending on its value

Errors

  • If there is an error in a piece of code, the program will stop running or get stuck
  • Can be as small as a typo or as big as a major flaw in logic
  • Syntax errors result when the "rules of coding" are not followed
  • Semantic errors result when the program does not follow the logic the programmer intended

Documentation

  • Available online
  • Equivalent to a user manual
  • Guidelines on how to use keywords and for what they are used
